
# Day 2 - Salesforce Platform Basics

## 1. What is Salesforce Platform?

Salesforce Platform is a cloud-based platform that helps businesses build applications, manage customer data, automate workflows, and customize business processes.

It allows developers and administrators to create apps, store data, and extend functionality using configuration tools and coding.

---

# 2. Salesforce Concepts

## App

An App in Salesforce is a collection of tabs, objects, and tools created for a specific business purpose.

Examples:
- Sales App
- Service App
- College Admission App

---

## Object

An Object is used to store data in Salesforce.

Objects are similar to database tables.

Examples:
- Account
- Contact
- Student
- Course

---

## Tab

A Tab is the user interface used to access objects and records in Salesforce.

Tabs help users easily navigate through the application.

---

# 3. CRM Concepts in Salesforce Platform

CRM concepts like Account, Contact, and Opportunity fit into Salesforce as standard objects inside Apps.

- Account stores company or organization information.
- Contact stores details about people associated with the account.
- Opportunity stores potential business deals or processes.

These objects work together inside Salesforce Apps to manage customer relationships and business workflows.

---

# 4. Configuration vs Coding

## Configuration (No Code)

Configuration means customizing Salesforce using clicks instead of writing code.

### Examples:
1. Creating custom fields
2. Building workflows using Flow Builder

### When to Use Configuration:
- Simple automation
- Quick customization
- Standard business requirements

---

## Coding (Apex)

Coding means developing advanced functionality using Apex and Lightning components.

### Examples:
1. Payment gateway integration
2. Complex approval systems

### When to Use Coding:
- Complex business logic
- Advanced automation
- External integrations

# 6. Multi-Tenant Architecture

Salesforce uses multi-tenant architecture, which means multiple companies share the same infrastructure while keeping their data secure and separate.

This helps Salesforce provide scalable and cost-effective services.

---

# 7. Difference Between App and Object

| App | Object |
|-----|--------|
| Collection of features and tabs | Stores data |
| Used for specific business purpose | Similar to database table |
| Contains multiple objects | Contains records and fields |

---

# 8. How Salesforce Allows Developers to Extend Functionality

Salesforce allows developers to extend functionality using:

- Apex Programming Language
- APIs
- Lightning Components
- Custom Objects
- Automation Tools

These tools help businesses build customized solutions based on their needs.
